# Documentation Guidelines

AI Knowledge Dashboard のドキュメントを整理・拡張する際は、以下の方針を必ず守る。

## 1. 孤立させない

- すべてのドキュメントは最低1か所からリンクされるようにする。
- README か docs/ 内のハブページからリンクされているかを作成直後に確認する。

## 2. The truth is in the code

- ドキュメントには仕様や目的、意思決定の背景を中心に書き、詳細実装やコード断片は必要最小限に抑える。
- 具体実装が必要になった場合も、擬似コードや図を活用し、コード断片のコピーペーストを避ける。

## 3. Mermaid の積極活用

- フロー/関連を説明する際はテキストだけにせず、Mermaid 図（sequence / flowchart / er / etc.）を使い視覚的に把握しやすくする。
- 図が冗長化しないよう、1ドキュメントあたり 1〜2 個に収め、要点にフォーカスした図解を心掛ける。

## 4. 参照しやすい構造

- docs/ 配下は役割ごとにサブディレクトリを切り、README から辿れるようにする。
- 新規ドキュメントを作成したら README の「Documentation」節や関連ドキュメントから相互リンクを追加する。
